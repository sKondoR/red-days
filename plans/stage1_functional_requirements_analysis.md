# Анализ функциональных требований - Этап 1

## 1. Основной экран с календарем

### Требование:
Приложение должно предоставлять основной экран с календарем, используя библиотеку react-native-calendars

### Подробный анализ:
- Необходимо реализовать календарь с возможностью визуализации дат
- Календарь должен поддерживать навигацию между месяцами
- Должна быть возможность отображения специальных маркеров для различных типов дней (менструация, овуляция, предполагаемые дни)
- Календарь должен быть интерактивным для добавления/редактирования информации о цикле

## 2. Отметка дней менструации

### Требование:
Пользователь должен иметь возможность отмечать дни менструации в календаре

### Подробный анализ:
- Реализовать возможность выбора даты и установки типа дня (менструация, овуляция, предменструальный синдром и т.д.)
- Обеспечить визуальное выделение отмеченных дней в календаре
- Реализовать возможность редактирования и удаления уже отмеченных дней
- Сохранять информацию о помеченных днях в локальной базе данных

## 3. Система уведомлений

### Требование:
Система должна отправлять уведомления для предупреждения о наступлении следующих дней

### Подробный анализ:
- Реализовать механизм планирования уведомлений на основе прогнозируемых дат
- Поддерживать различные типы уведомлений (начало менструации, овуляция, напоминание о записи)
- Обеспечить настраиваемость параметров уведомлений (время, частота, типы)
- Использовать react-native-push-notification для реализации уведомлений

## 4. Статистика и аналитика цикла

### Требование:
Приложение должно предоставлять статистику и аналитику по менструальному циклу

### Подробный анализ:
- Рассчитывать среднюю продолжительность цикла на основе введенных данных
- Определять закономерности в продолжительности фаз цикла
- Предоставлять визуализацию данных (графики, диаграммы)
- Отображать прогнозируемые даты следующих фаз цикла
- Реализовать историю циклов за несколько месяцев

## 5. Хранение данных локально

### Требование:
Данные должны храниться локально (SQLite для сложных данных, AsyncStorage для настроек)

### Подробный анализ:
- Использовать SQLite для хранения информации о цикле, датах, симптомах
- Использовать AsyncStorage для хранения настроек приложения (настройки уведомлений, тема и т.д.)
- Обеспечить безопасность хранения чувствительных данных
- Реализовать механизм резервного копирования и восстановления данных
- Обеспечить миграцию данных при обновлениях приложения
