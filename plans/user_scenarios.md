# Пользовательские сценарии использования приложения

## Сценарий 1: Первый запуск приложения

**Цель**: Пользователь открывает приложение впервые и начинает его использовать

**Шаги**:
1. Пользователь запускает приложение
2. Приложение показывает приветственный экран или экран настроек
3. Пользователь может просмотреть краткое руководство или сразу перейти к календарю
4. Пользователь начинает отмечать свои первые данные о цикле

## Сценарий 2: Отметка дня менструации

**Цель**: Пользователь отмечает день менструации в календаре

**Шаги**:
1. Пользователь открывает главный экран с календарем
2. Пользователь выбирает текущую дату или прошедшую дату
3. Пользователь нажимает на кнопку "Добавить запись" или просто нажимает на дату
4. Открывается экран добавления/редактирования записи
5. Пользователь указывает интенсивность, симптомы, добавляет заметки
6. Пользователь сохраняет запись
7. Запись отображается в календаре соответствующим образом

## Сценарий 3: Просмотр статистики цикла

**Цель**: Пользователь хочет получить информацию о своих циклах

**Шаги**:
1. Пользователь переходит на экран статистики
2. Пользователь видит общую информацию: среднюю длительность цикла, продолжительность последнего цикла
3. Пользователь может видеть прогноз следующего цикла
4. Пользователь просматривает графики и диаграммы для анализа тенденций

## Сценарий 4: Получение уведомления

**Цель**: Пользователь получает уведомление о предстоящем цикле

**Шаги**:
1. Система автоматически рассчитывает дату следующего цикла на основе истории
2. За определённое время до начала (например, за 24 часа) система отправляет уведомление
3. Пользователь получает уведомление даже если приложение не запущено
4. Пользователь может открыть приложение прямо из уведомления

## Сценарий 5: Настройка уведомлений

**Цель**: Пользователь изменяет параметры уведомлений

**Шаги**:
1. Пользователь переходит в настройки уведомлений
2. Пользователь включает или отключает различные типы уведомлений
3. Пользователь устанавливает время получения уведомлений
4. Пользователь сохраняет изменения

## Сценарий 6: Экспорт данных

**Цель**: Пользователь хочет экспортировать свои данные

**Шаги**:
1. Пользователь переходит в настройки приложения
2. Пользователь находит раздел "Экспорт данных"
3. Пользователь выбирает формат экспорта (JSON или CSV)
4. Пользователь инициирует экспорт
5. Данные сохраняются в файл, доступный для скачивания или отправки

## Сценарий 7: Просмотр подробной информации о конкретном дне

**Цель**: Пользователь хочет посмотреть или отредактировать информацию о конкретном дне

**Шаги**:
1. Пользователь находит интересующую дату в календаре
2. Пользователь нажимает на дату
3. Открывается экран просмотра дня с подробной информацией
4. Пользователь может редактировать запись или просто просмотреть её

## Сценарий 8: Регулярное использование приложения

**Цель**: Пользователь регулярно использует приложение для отслеживания цикла

**Шаги**:
1. Пользователь ежедневно открывает приложение
2. Пользователь отмечает свои симптомы и состояние
3. Пользователь просматривает календарь для понимания текущего положения в цикле
4. Пользователь периодически проверяет статистику для анализа изменений
